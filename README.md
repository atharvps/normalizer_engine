## вңҰ What is This?

**Normalizer Engine** is a browser-based tool that transforms any **Context-Free Grammar (CFG)** into either **Chomsky Normal Form (CNF)** or **Greibach Normal Form (GNF)** вҖ” one transparent step at a time.

No installations. No back-end. No black boxes. Just paste your grammar, hit a button, and *watch the theory come to life*.

---

## вңҰ Features

| Feature | Description |
|---|---|
| рҹ”Ғ **CNF Conversion** | Full 7-step pipeline: start isolation вҶ’ Оө-elimination вҶ’ unit rules вҶ’ useless symbols вҶ’ terminal isolation вҶ’ binarization |
| рҹ”Ғ **GNF Conversion** | 9-step pipeline including variable ordering, direct/indirect left-recursion resolution via Zi substitution, and backward substitution |
| рҹ§© **Step-by-Step Mode** | Advance one transformation at a time with the **Next вҶ’** button |
| вҡЎ **Run All Mode** | Auto-advance through all steps with animated timing |
| рҹҺЁ **Syntax Highlighting** | Color-coded non-terminals, terminals, and newly introduced symbols |
| рҹҢ— **Dark / Light Theme** | One-click toggle вҖ” dark (teal-green) and light (aqua-teal) modes |
| Оө **Epsilon Insertion** | Click the Оө button to insert the epsilon character directly into the textarea |
| рҹ“җ **IDE-style Output** | Grammar renders in a code-editor window with animated rule rows |
| вҷ»пёҸ **Reset** | Clear state and start fresh without refreshing the page |

## вңҰ Grammar Input Format

Write your grammar using the following conventions:

```
S -> A B C | eps
A -> a A | eps
B -> b B | eps
C -> c C | c
```

| Rule | Convention |
|---|---|
| **Non-terminals** | Single uppercase letter: `S`, `A`, `B` |
| **Terminals** | Lowercase letters or digits: `a`, `b`, `0` |
| **Productions** | Separated by pipe `\|` |
| **Epsilon** | Write `eps` or click the **Оө** button to insert `Оө` directly |
| **Arrow** | Use `->` or `вҶ’` |
| **First Rule** | The LHS of the first rule is treated as the start symbol |

### Example Grammars

**Balanced parentheses:**
```
S -> ( S ) | eps
```

**Simple arithmetic:**
```
E -> E + T | T
T -> T * F | F
F -> ( E ) | a
```

**Left-recursive (GNF will resolve this):**
```
A -> A a | b
```

## вңҰ CNF Transformation Pipeline

Chomsky Normal Form requires every rule to be of the form `A вҶ’ BC` or `A вҶ’ a`.

```
Input Grammar
     в”Ӯ
     в–ј
[Step 1] Parse & Validate
     в”Ӯ
     в–ј
[Step 2] Isolate Start Symbol
         Add S0 вҶ’ S if S appears on any RHS
     в”Ӯ
     в–ј
[Step 3] Eliminate Оө-Productions
         Find all nullable variables; expand combinations
     в”Ӯ
     в–ј
[Step 4] Eliminate Unit Rules
         Replace A вҶ’ B chains with their transitive closures
     в”Ӯ
     в–ј
[Step 5] Eliminate Useless Symbols
         Remove non-generating & unreachable symbols
     в”Ӯ
     в–ј
[Step 6] Isolate Terminals
         Wrap terminals in new rules: T_a вҶ’ a
     в”Ӯ
     в–ј
[Step 7] Binarize Long Rules
         Break A вҶ’ B C D into A вҶ’ B X_1, X_1 вҶ’ C D
     в”Ӯ
     в–ј
  вң“ CNF
```

## вңҰ GNF Transformation Pipeline

Greibach Normal Form requires every rule to start with a terminal: `A вҶ’ a Оұ`.

```
Input Grammar
     в”Ӯ
     в–ј
[Steps 1вҖ“5] Full CNF Preparation (same as above)
     в”Ӯ
     в–ј
[Step 6] Order Variables
         Assign indices AвӮҒ, AвӮӮ, ... AвӮҷ to enforce ordering
     в”Ӯ
     в–ј
[Step 7] Resolve Left Recursion
         For each Aбөў вҶ’ Aвұј Оұ where j вүӨ i:
           Substitute and eliminate using Zбөў variables
     в”Ӯ
     в–ј
[Step 8] Backward Substitution
         Work backwards so every rule starts with a terminal
     в”Ӯ
     в–ј
[Step 9] Wrap Remaining Terminals
         Introduce X_a вҶ’ a for terminals in longer rules
     в”Ӯ
     в–ј
  вң“ GNF
```
